# Privacy Policy - Azure DevOps PR Agent Extension

**Last Updated:** February 1, 2026 
**Version:** 3.12.0

## Overview

The Azure DevOps PR Agent Extension ("the Extension") is designed to enhance your development workflow by providing AI-powered analysis of pull requests in Azure DevOps. This Privacy Policy explains how we collect, use, and protect your information when you use our Extension.

## Information We Collect

### 1. Pull Request Data
- **Source Code**: The Extension accesses pull request content, including modified files, code changes, and commit messages to perform AI analysis.
- **Metadata**: Pull request titles, descriptions, author information, branch names, and related work items.
- **Comments**: Existing comments and discussions on pull requests for context analysis.

### 2. Configuration Data
- **Settings**: Your Extension configuration preferences, API endpoints, and analysis options.
- **Authentication**: Azure DevOps Personal Access Tokens (stored locally in your browser).

### 3. Usage Analytics
- **Performance Metrics**: Analysis execution times, success/failure rates, and feature usage statistics.
- **Error Logs**: Technical error information for debugging and improvement purposes.

## How We Use Your Information

### 1. Core Functionality
- **AI Analysis**: Send pull request data to configured AI services (OpenAI/Azure OpenAI) for code review, improvement suggestions, and compliance checks.
- **Results Generation**: Process AI responses to generate formatted comments and suggestions.
- **Auto-Approval**: Analyze pull request characteristics to determine auto-approval eligibility.

### 2. Service Improvement
- **Performance Optimization**: Use aggregated usage data to improve Extension performance and reliability.
- **Feature Development**: Analyze usage patterns to develop new features and enhancements.

## Data Sharing and Third-Party Services

### 1. AI Service Providers
- **OpenAI/Azure OpenAI**: Pull request content is sent to AI services for analysis. This data is processed according to the AI provider's privacy policy.
- **Data Transmission**: All data transmission to AI services is encrypted using HTTPS/TLS.
- **Retention**: AI service providers may temporarily store data for processing but do not retain it for training purposes (when using Azure OpenAI with appropriate settings).

### 2. Microsoft Azure DevOps
- **Integration**: The Extension operates within the Azure DevOps environment and follows Microsoft's security and privacy standards.
- **Data Access**: Only accesses data that you have permission to view within your Azure DevOps organization.

## Data Security

### 1. Encryption
- **In Transit**: All data transmission is encrypted using industry-standard TLS/HTTPS protocols.
- **At Rest**: Configuration data is stored securely in your browser's local storage.

### 2. Access Control
- **Authentication**: Uses Azure DevOps authentication mechanisms and Personal Access Tokens.
- **Permissions**: Respects Azure DevOps permission models and only accesses authorized resources.

### 3. Data Minimization
- **Scope Limitation**: Only processes data necessary for the requested analysis.
- **Temporary Processing**: Data is processed temporarily and not stored permanently by the Extension.

## Your Rights and Choices

### 1. Data Control
- **Configuration**: You control which analyses are performed and what data is processed.
- **Opt-Out**: You can disable or uninstall the Extension at any time.
- **Selective Use**: Choose which pull requests to analyze and which features to use.

### 2. Access and Deletion
- **Local Data**: You can clear Extension settings and cached data through your browser settings.
- **Azure DevOps Data**: Managed through your Azure DevOps organization's data policies.

## Data Retention

### 1. Extension Data
- **Configuration**: Stored locally until you clear browser data or uninstall the Extension.
- **Temporary Data**: Processing data is not retained after analysis completion.

### 2. Third-Party Retention
- **AI Services**: Subject to the retention policies of your chosen AI service provider.
- **Azure DevOps**: Subject to Microsoft's Azure DevOps data retention policies.

## Children's Privacy

This Extension is not intended for use by individuals under the age of 13. We do not knowingly collect personal information from children under 13.

## International Data Transfers

Data may be transferred to and processed in countries other than your own, including the United States, where AI service providers operate. These transfers are conducted in accordance with applicable data protection laws.

## Compliance

### 1. Regulatory Compliance
- **GDPR**: For users in the European Union, we comply with General Data Protection Regulation requirements.
- **CCPA**: For California residents, we comply with California Consumer Privacy Act requirements.
- **Enterprise Compliance**: Supports enterprise compliance requirements through configurable data handling options.

### 2. Security Standards
- **Industry Standards**: Follows industry best practices for data security and privacy.
- **Regular Updates**: Security measures are regularly reviewed and updated.

## Changes to This Privacy Policy

- Providing notice through the Extension interface
- Publishing updates in the Extension marketplace listing
